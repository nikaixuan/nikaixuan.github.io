Algorithm

Sort algorithm

Insertion sort
From the second element to the last one, check the element with all previous elements and insert it to the right place. So the first ith elements are already sorted.
Stable/In place
Worst case: Desc O(n^2)
Already sorted: O(n)

Use for small or partially-sorted arrays.

Selection sort
In the outer loop, find the smallest element everytime and put it at the front of the array. In other word, find the smallest element from i to n and put it to i.
Not stable/In place
All the case: O(n^2)

Bubble sort
Always move the largest element towards the end of the array. Need n^2 times swap. Like bubble from buttom to the top.
Stable/In place
All the case: O(n^2)

Shell sort
Set an initial gap n (usually length/3+1). Then have n subarray to use insertion sort, and minimize the gap (n/3+1)… finally get the sorted array.
每次对子数组的插入排序都会使数组逐渐变得有序
Not stable/In place
Worst case: O(n^2)
Already sorted: O(nlogn)

Quick sort
Most used algorithm, recursive.
Process:

  1. The input contains the array, the left index and the right index.
  2. Use right < left to stop the recursion, use the partition method to get the partition base, then call the sort method recursively using (left, base-1) and (base+1, right).
  3. In partition method, set the left as the base_index, array[left] as base, find the position of the base element and swap the base element to the correct position. The index of the position is the partition index in the new recursion.
    Not stable/In place
    Worst when already sorted: O(n^2)
    Average: O(nlogn)

Merge sort
Like decision tree. Just keep split origin array into two pieces, then merge.
When merging, use a copy of origin array, use two pointer to swap the element.
Stable
Space complexity: O(n)
Time complexity: O(nlgn) in all the cases.
