Posted on 2019-06-01

Data Structures

1.Array

Optimal for indexing, bad at searching, inserting and deleting.

Dynamic array Have additional space for extra elements.

Indexing: Linear array: O(1), Dynamic array: O(1)
Search: Linear array: O(n), Dynamic array: O(n)
Optimized Search(Binary search): Linear array: O(log n), Dynamic array: O(log n)
Insertion and deletion: Linear array: O(n), Dynamic array: O(n)

熟悉Arrays工具类的方法如equals, sort, asList.
熟悉binary search.

2.Linked list

Store data with node, every node has reference to next node.
Optimal for insertion, deletion, bad at indexing and searching.

Stack
Last in first out.

Queue
First in first out.

Doubly linked list
Every node has two references, prev and next.

Indexing: O(n)
Search: O(n)
Optimized Search: O(n)
Insertion and deletion: O(1)

Find cycle in linked list(Two pointer)

Reverse linked list

Find middle of linked list(Two pointer)
Build queue/stack use stack/queue.

3.HashTable/HashMap

Combine of array and linked list.
Hash collision is when two keys have same hash value output.

If the hash function is valid and use open hashing.
Searching: O(1)
Insertion and deletion: O(1)
In Java, use close hashing, worst case: O(n)

4.Tree

Binary search tree
Tree like data structure and every node has two children.
Left child is smaller and right key is larger.

Indexing: O(logn)
Searching: O(logn)
Insertion and deletion: O(logn)
Worst case: O(n), when the compiler uses some optimization, the complexity is O(logn)

Preoeder: mid-left-right
Inorder: left-mid-right
Postorder: left-right-mid

AVL tree
Self-balancing search tree.
Red-Black tree
B-Tree
